# CLAUDE.md — Проект «Спортивные судьи»

## Что это за проект

Flask веб-приложение для ведения базы данных спортивных судей по спортивному туризму (Московская область). Для каждого судьи ведётся электронная карточка учёта судейской деятельности: личные данные, история присвоения квалификационных категорий, прохождение теоретической подготовки и сдача квалификационного зачёта, практика судейства официальных соревнований. По данным карточки формируется официальный документ Word — «Карточка учёта судейской деятельности спортивного судьи» — в точном соответствии с образцом из `data/`.

## Стек

- **Python 3.11+**
- **Flask** — веб-фреймворк
- **SQLAlchemy** — ORM для SQLite
- **python-docx** — генерация итогового документа Word по образцу
- **Jinja2** — шаблоны (входит в Flask)
- **SQLite** — база данных (файл `athletes.db`)

## Модели базы данных

### Settings (одна запись — общая информация об организации для шапки карточки)

| Поле | Тип | Описание |
|---|---|---|
| id | Integer PK | |
| sport_name | String | Наименование вида спорта |
| sport_code | String | Номер-код вида спорта |
| organization_name | String | Наименование организации, осуществляющей учёт судейской деятельности |
| organization_address | String | Адрес (место нахождения) организации |
| organization_contacts | String | Телефон, адрес электронной почты организации |

### Judge (спортивный судья)

| Поле | Тип | Описание |
|---|---|---|
| id | Integer PK | |
| last_name / first_name / middle_name | String | ФИО |
| birth_date | Date | Дата рождения |
| region | String | Субъект Российской Федерации |
| municipality | String | Муниципальное образование |
| rank | String | Спортивное звание в данном виде спорта (при наличии) |
| education | String | Образование |
| workplace | String | Место работы (учёбы), должность |
| contacts | String | Контактные телефоны, адрес электронной почты |
| judging_start_date | Date | Дата начала судейской деятельности |
| is_active | Boolean | Активен в текущем списке (default: True) |
| created_at / updated_at | DateTime | |

Текущая квалификационная категория судьи (`current_category`) — не хранимое поле, а свойство: категория из самой последней по дате записи `JudgeCategoryRecord`. Именно по ней работают фильтры ССВК/СС1К/СС2К/СС3К на странице списка судей.

### JudgeCategoryRecord (история квалификационных категорий, 1 запись = 1 строка карточки)

category, action («присвоена»/«подтверждена»/«лишена»/«восстановлена»), date, document_number, issuing_organization, signed_by, record_keeper.

### JudgeTrainingRecord (прохождение теоретической подготовки, сдача квалификационного зачёта)

seminar_name, seminar_date, organizer, location, participant_category/score/date (в качестве участника), lecturer_category/score/date (в качестве лектора), exam_protocol_number, exam_score, record_date, record_keeper.

Даты семинаров хранятся строкой (в образцах встречаются диапазоны вида «09-10.11.2019г.»), не `Date`.

### JudgeCompetitionRecord (практика судейства официальных спортивных соревнований)

event_date, location, judge_position, competition_name, competition_status, score, record_date, record_keeper.

`competition_name` (полное наименование соревнования с видом программы) и `competition_status` (официальный статус: «Чемпионат субъекта Российской Федерации», «Первенство России» и т.п.) — два разных поля, в документе выводятся двумя отдельными строками одной ячейки.

## Маршруты (routes/judges.py)

- `GET /judges/` → список активных судей + кнопки-фильтры ССВК/СС1К/СС2К/СС3К
- `GET /judges/vsk`, `/1k`, `/2k`, `/3k` → судьи данной категории (по `current_category`), включая исключённых — активные подсвечены
- `GET|POST /judges/new`, `GET /judges/<id>`, `GET|POST /judges/<id>/edit`
- `POST /judges/<id>/activate|deactivate|delete`
- `GET /judges/<id>/export` → генерация и скачивание карточки судьи в формате .docx
- CRUD для истории: `/judges/<id>/category-records`, `/judges/category-records/<id>/edit|delete` (и аналогично `training-records`, `competition-records`) — редактирование прямо на странице карточки судьи, произвольное число строк

`GET|POST /settings` (в `app.py`) — общая информация об организации.

## Генерация Word-документа (judge_docx_generator.py)

Документ **не строится с нуля** — вместо этого открывается образец `data/Учетная карточка судьи СС1К Серганова СА.docx` как шаблон, в нём находятся и заполняются нужные ячейки, а строки трёх повторяющихся таблиц (история категорий, подготовка, практика судейства) клонируются по количеству реальных записей судьи. Это сохраняет исходное форматирование, объединение ячеек и границы один в один.

Особенность вёрстки образца: у таблицы «подготовка» каждая запись занимает **3 физические строки** (первые 12 столбцов объединены по вертикали, последний столбец — дата внесения записи / пустая строка под подпись / ФИО ответственного — три отдельные незамёрженные ячейки). У таблицы «практика судейства» — **2 физические строки** на запись (аналогично: 4 общих столбца объединены, последний столбец — дата записи / ФИО ответственного). У таблицы истории категорий — 1 строка на запись, без объединений. `_replace_repeating_rows()` в `judge_docx_generator.py` реализует это клонирование по группам.

Документ «Представление к присвоению квалификационной категории» (образец `data/Представление СС1К Серганов СА.docx`) — генерация из приложения пока **не реализована** (отдельная задача на будущее).
