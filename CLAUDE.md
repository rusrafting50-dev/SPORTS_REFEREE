# CLAUDE.md — Проект Сборная команда

## Что это за проект

Flask веб-приложение для ведения базы кандидатов в спортивную сборную команду Московской области по спортивному туризму. Заменяет ручное ведение Excel-файлов: данные хранятся в SQLite, а итоговые документы генерируются автоматически в точном соответствии с официальной формой.

---

## Стек

- **Python 3.11+**
- **Flask** — веб-фреймворк
- **SQLAlchemy** — ORM для SQLite
- **openpyxl** — чтение и генерация Excel-файлов
- **Jinja2** — шаблоны (входит в Flask)
- **SQLite** — база данных (файл `athletes.db`)

## Справочники (references.py)

Все значения для выпадающих списков хранятся в `references.py` как константы. Источник — лист «Не удалять» из исходного Excel.

- `DISCIPLINES` — список дисциплин спортивного туризма
- `CATEGORIES` — спортсмен, тренер, специалист и др.
- `AGE_CATEGORIES` — мужчины, женщины, юниоры, юниорки, юноши, девушки, мальчики, девочки
- `RANKS` — ЗТР, ЗМС, МСМК, МС, КМС, I, II, III, I юн, II юн, III юн
- `TERRITORIES` — города и районы Московской области
- `RESULT_CODES_REGIONAL` — ЧМО-1..ЧМО-16, КМО-1..КМО-14, ПМО-1..
- `RESULT_CODES_NATIONAL` — ЧР-1..ЧР-30, КР-1..
- `RESULT_CODES_INTERNATIONAL` — ЧМ-1.., ЧЕ-1.., ЦФО-1..
- `NATIONAL_TEAM_ROLES` — Основной состав (мужчины, женщины), Резервный состав и т.д.

## Маршруты (routes)

### athletes.py
- `GET /` → редирект на `/athletes`
- `GET /athletes` → список с фильтрами
- `GET /athletes/new` → форма добавления
- `POST /athletes/new` → сохранение нового спортсмена
- `GET /athletes/<id>` → карточка спортсмена
- `GET /athletes/<id>/edit` → форма редактирования
- `POST /athletes/<id>/edit` → сохранение изменений
- `POST /athletes/<id>/deactivate` → исключить из активного списка
- `POST /athletes/<id>/activate` → вернуть в активный список

### import_export.py
- `GET /import` → страница импорта
- `POST /import/upload` → загрузка Excel-файла и парсинг в базу
- `GET /import/changes` → страница импорта файла изменений
- `POST /import/changes/upload` → парсинг файла изменений (Включить/Исключить)

### reports.py
- `GET /reports` → страница настройки фильтров
- `POST /reports/preview` → предпросмотр списка в браузере
- `POST /reports/export` → генерация и скачивание .xlsx

## Фильтры для формирования отчёта

На странице `/reports` пользователь выбирает:
- **Дисциплина** — одна или несколько из справочника
- **Пол** — все / мужской / женский
- **Возрастная категория** — одна или несколько
- **Разряд** — один или несколько
- **Организация** — фильтр по тексту или из списка
- **Территория** — один или несколько районов
- **Категория** — спортсмен / тренер / специалист / все
- **Состав сборной РФ** — только включённые в сборную / все
- **Дата документа** — проставляется в шапку итогового файла

Результат — отфильтрованный список, который можно просмотреть в браузере и выгрузить в Excel.

## Генерация Excel (excel_generator.py)

Итоговый документ должен точно соответствовать форме официального списка:

**Шапка:**
- Название организации
- Должность и ФИО председателя, главного тренера
- Заголовок: «СПИСОК кандидатов в спортивную сборную команду Московской области»
- Вид спорта и год
- Счётчики: Всего человек / тренеры / специалисты / спортсмены

**Таблица данных** (15 столбцов, нумерация с 1):
1. № п/п
2. Фамилия, имя, отчество
3. Дата рождения
4. Пол
5. Спортивная дисциплина
6. Категория
7. Возрастная категория (ЕВСК)
8. Спортивное звание, разряд
9. Организация
10. Территориальная принадлежность
11. Личный тренер
12. Результат — Московские областные
13. Результат — Всероссийские
14. Результат — Международные
15. Состав сборной РФ

**Форматирование:**
- Шрифт Times New Roman, размер 10
- Заголовки столбцов — жирные, выравнивание по центру
- Перенос текста в ячейках
- Границы таблицы
- Ширина столбцов — как в оригинале

Также поддерживается генерация документа **«Изменения в список»** — с разделами «Исключить» и «Включить».

## Импорт из Excel

`import_export.py` должен уметь:

1. **Импорт основного списка** (`Список_кандидатов_*.xlsx`):
   - Пропустить строки шапки (до строки с «№ п/п»)
   - Парсить данные начиная со строки с «1»
   - Дата рождения хранится как Excel serial number → конвертировать в Python `date`
   - Если спортсмен с таким ФИО + датой рождения уже есть в базе — обновить, не дублировать

2. **Импорт изменений** (`Изменения_*.xlsx`):
   - Определить раздел «Исключить» / «Включить» по заголовку строки
   - Для «Исключить»: найти спортсмена в базе и установить `is_active = False`, добавить запись в ChangeLog
   - Для «Включить»: добавить спортсмена или активировать существующего, добавить запись в ChangeLog

## Интерфейс

Минималистичный, функциональный. Используй Bootstrap 5 (CDN).

- Навигация: Спортсмены | Импорт | Отчёты | Настройки
- Таблица спортсменов: сортировка по столбцам, фильтры сверху
- Формы: выпадающие списки для всех справочников, поля с автодополнением для тренеров и организаций

## Важные особенности

- Даты рождения в исходных Excel хранятся как числа (Excel serial date) — нужна конвертация через `datetime(1899, 12, 30) + timedelta(days=n)`
- Одна дисциплина может содержать несколько значений через запятую («Дистанции пешеходные, дистанции лыжные») — хранить как строку, не нормализовывать
- Поле «Результат» тоже может быть составным («КР-1, ЧР-2») — хранить как строку
- Номера строк в итоговом Excel должны сквозные и начинаться с 1
- Счётчики в шапке (тренеры / специалисты / спортсмены) вычисляются из отфильтрованных данных автоматически
