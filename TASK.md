# TASK.md — Задание на разработку: АтлетБаза

## Контекст

Нужно разработать Flask веб-приложение для ведения базы кандидатов в спортивную сборную команду Московской области по спортивному туризму.

Сейчас этот учёт ведётся вручную в Excel. Два образца реальных документов лежат в папке `data/`:
- `Список_кандидатов_в_сборную_команду_МО_спортивный_туризм_2026.xlsx` — основной список (~120 человек)
- `Изменения_списка_в_сборную_МО_спортивный_туризм_31.xlsx` — документ с изменениями (разделы «Исключить» / «Включить»)

Итоговое приложение должно заменить ручную работу с Excel и позволять генерировать официальные документы точно по установленной форме.

Полное описание архитектуры, моделей, маршрутов и требований — в файле `CLAUDE.md`.

---

## Задание

Реализуй приложение поэтапно, строго следуя `CLAUDE.md`. Не придумывай ничего от себя — только то, что описано.

## Этап 4 — Генерация Excel

- [ ] `excel_generator.py` — функция `generate_report(athletes, settings, doc_date)`:
  - Принимает список объектов `Athlete`, настройки из `ReportSettings`, дату документа
  - Возвращает объект `openpyxl.Workbook`
  - Шапка документа точно по форме (см. образец в `data/`)
  - Таблица с 15 столбцами, нумерация с 1
  - Строка заголовков столбцов — жирная, по центру, с переносом
  - Шрифт Times New Roman 10pt для данных
  - Границы таблицы у всех ячеек с данными
  - Счётчики (тренеры / специалисты / спортсмены / всего) вычислять автоматически
- [ ] Функция `generate_changes_report(exclude_list, include_list, settings, doc_date, doc_number)` — для документа «Изменения»
- [ ] Маршрут `POST /reports/export` — применяет фильтры, генерирует файл, отдаёт на скачивание
- [ ] Имя файла: `Список_МО_спортивный_туризм_{year}_{дата}.xlsx`

**Проверка:** сгенерированный файл открывается в Excel и визуально совпадает с образцом из `data/`

## Требования к коду

- Весь код на русском в комментариях там, где это помогает пониманию
- Обработка ошибок: если файл импорта не того формата — показать понятное сообщение
- Flash-сообщения для всех действий (добавлено / обновлено / ошибка)
- Не использовать `<form>` как HTML-тег напрямую без `method` и `action`
- Все справочники брать только из `references.py`, не хардкодить в шаблонах
